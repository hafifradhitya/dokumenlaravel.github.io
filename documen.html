Instalasi Laravel

# Membuat Proyek Laravel


Sebelum membuat proyek Laravel pertama Anda, pastikan mesin lokal Anda telah menginstal PHP dan Komposer. Jika Anda mengembangkan di macOS, PHP dan Composer dapat diinstal dalam hitungan menit melalui Laravel Herd. Selain itu, kami menyarankan untuk menginstal Node dan NPM.

composer create-project laravel/laravel example-app

laravel new example-app

Atau, Anda dapat membuat proyek Laravel baru dengan menginstal installer Laravel secara global melalui Composer:

composer global require laravel/installer

Setelah proyek dibuat, mulai server pengembangan lokal Laravel menggunakan perintah serve Laravel Artisan:

cd example-app

php artisan serve


Routing Laravel

# Dasar Routing


Rute Laravel paling dasar menerima URI dan penutupan, menyediakan metode yang sangat sederhana dan ekspresif untuk mendefinisikan rute dan perilaku tanpa file konfigurasi perutean yang rumit:

use Illuminate\Support\Facades\Route;

Route::get('/greeting', function () { return 'Hello World'; });

# File Route Default


Semua rute Laravel ditentukan dalam file rute Anda, yang terletak di direktori rute. File-file ini secara otomatis dimuat oleh App\Providers\RouteServiceProvider aplikasi Anda. File rute/web.php mendefinisikan rute untuk antarmuka web Anda. Rute ini ditetapkan ke grup middleware web, yang menyediakan fitur seperti status sesi dan perlindungan CSRF. Rute diroutes/api.php tidak memiliki kewarganegaraan dan ditugaskan ke grup api middleware. Untuk sebagian besar aplikasi, Anda akan mulai dengan menentukan rute di file rute/web.php Anda. Rute yang ditentukan di rute/web.php dapat diakses dengan memasukkan URL rute yang ditentukan di browser Anda. Misalnya, Anda dapat mengakses rute berikut dengan membuka http://example.com/user di browser Anda:

use App\Http\Controllers\UserController;

Route::get('/user', [UserController::class, 'index']);


Controller

# Dasar Controllers


Untuk membuat controller baru dengan cepat, Anda dapat menjalankan perintah make:controller Artisan. Secara default, semua pengontrol untuk aplikasi Anda disimpan di direktori app/Http/Controllers:

php artisan make:controller UserController

Mari kita lihat contoh controllers dasar. Controllers mungkin memiliki sejumlah metode publik yang akan merespons permintaan HTTP yang masuk:

<?php
namespace App \\ Http \\ Controllers ;
use App \\ Models \\ User ;
use Illuminate \\ View \\ View ;
class UserController extends Controller {

    /**
* Show the profile for a given user.
*/
public function show ( string $id ) : View {

        return view ( 'user.profile' , [
            'user' => User::findOrFail($id)
        ] ) ;
}
}

Setelah Anda menulis kelas dan metode controllers Anda dapat menentukan rute ke metode controllers seperti:

use App\Http\Controllers\UserController;
Route::get('/user/{id}', [UserController::class, 'show']);


Views

# Membuat dan Merender View


Anda dapat membuat View dengan menempatkan file dengan ekstensi .blade.php di direktori resource/views aplikasi Anda atau dengan menggunakan perintah make:view Artisan:

php artisan make:view View

Ekstensi .blade.php menginformasikan kerangka kerja bahwa file tersebut berisi template Blade. Templat Blade berisi HTML serta arahan Blade yang memungkinkan Anda mengulangi nilai dengan mudah, membuat pernyataan "jika", mengulangi data, dan banyak lagi.

Route::get('/', function () {
  view('greeting', ['name' => 'James']);
});

View juga dapat dikembalikan menggunakan View fasad:

use Illuminate\Support\Facades\View;
return View::make('greeting', ['name' => 'James']);

Seperti yang Anda lihat, argumen pertama yang diteruskan ke view helper sesuai dengan nama file view di direktori resources/views. Argumen kedua adalah array data yang harus tersedia untuk View. Dalam hal ini, kita meneruskan variabel nama, yang ditampilkan dalam View menggunakan sintaks Blade.

# Directori View Bersarang


View juga dapat disarangkan dalam subdirektori dari direktori resources/views. Notasi "Titik" dapat digunakan untuk mereferensikan View bertingkat. Misalnya, jika View Anda disimpan di resources/views/admin/profile.blade.php, Anda dapat mengembalikannya dari salah satu rute/pengontrol aplikasi Anda seperti:

return view('admin.profile', $data);


Models

Menghasilkan CLass Model


Untuk memulai, mari buat model Eloquent. Model biasanya berada di direktori app\Models dan memperluas kelas Illuminate\Database\Eloquent\Model. Anda dapat menggunakan perintah make:model Artisan untuk menghasilkan model baru:

php artisan make:model Flight

Jika Anda ingin membuat migrasi database saat membuat model, Anda dapat menggunakan opsi --migration atau -m:

php artisan make:model Flight --migration

Anda dapat membuat berbagai tipe kelas lainnya saat membuat model, seperti pabrik, seeder, kebijakan, pengontrol, dan permintaan formulir. Selain itu, opsi berikut dapat digabungkan untuk membuat beberapa kelas sekaligus:

# Hasilkan model dan FlightFactory class...
php artisan make:model Flight --factory


php artisan make:model Flight -f


# Hasilkan model dan FlightSeeder

class...
php artisan make:model Flight --seed


php artisan make:model Flight -s


# Hasilkan model dan FlightController

class...
php artisan make:model Flight --controller


php artisan make:model Flight -c


# Hasilkan model, FlightController resource

class, and form request classes...
php artisan make:model Flight --controller --

resource --requests
php artisan make:model Flight -crR


# Hasilkan model dan FlightPolicy

class...
php artisan make:model Flight --policy


# Hasilkan model dan migration, factory, seeder, and controller...

php artisan make:model Flight -mfsc


# Shortcut to generate a model,

migration, factory, seeder, policy, controller, and form requests...
php artisan make:model Flight --all


# Hasilkan pivot model...
php artisan make:model Member --pivot


php artisan make:model Member -p
